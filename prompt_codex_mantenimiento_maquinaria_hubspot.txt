Antes de escribir código, lee primero el archivo:

contexto_mantenimiento_maquinaria_hubspot.txt

Ese archivo contiene el contexto funcional, reglas de negocio, arquitectura esperada y consideraciones importantes del proyecto.

Tu tarea es crear el proyecto completo en Node.js para implementar la API descrita en el archivo de contexto.

Objetivo:
Crear una API sencilla que reciba un dealId desde un workflow de HubSpot, consulte los line items del negocio para obtener el código de máquina, busque ese código en todas las hojas de un Google Sheet y cree line items de mantenimiento asociados al deal según los productos y frecuencias marcadas con X en la hoja correspondiente.

Requisitos principales:

1. Crear un proyecto Node.js con Express.
2. Implementar el endpoint:

POST /api/maintenance/generate-line-items

Body:

{
  "dealId": "123456789"
}

3. El endpoint solo debe recibir dealId.
4. El código de máquina debe obtenerse desde los line items asociados al deal en HubSpot.
5. Leer Google Sheets usando la API oficial de Google.
6. Recorrer todas las hojas del Google Sheet.
7. Buscar coincidencia del código de máquina en cada hoja.
8. Cuando se encuentre la hoja correcta, leer las filas de productos.
9. Detectar dinámicamente las columnas:
   - DESCRIPCION
   - # PARTE
   - 250
   - 500
   - 1000
   - 2000
   - 3000
   - 5000
10. Por cada fila válida, buscar el producto en HubSpot por el número de parte.
11. Crear un line item asociado al deal.
12. Marcar las propiedades de frecuencia como "Si" cuando la celda tenga una X.
13. Marcar las propiedades de frecuencia como "No" cuando la celda no tenga X.
14. Evitar duplicados cuando el workflow se ejecute más de una vez.
15. Devolver un resumen del proceso.
16. Crear README.md y .env.example.

Arquitectura obligatoria:
Usa una arquitectura monolítica sencilla. No implementes arquitectura hexagonal ni patrones innecesariamente complejos.

Consideraciones técnicas:

- Usa dotenv para variables de entorno.
- Usa googleapis para Google Sheets.
- Puedes usar axios o @hubspot/api-client para HubSpot, pero deja el código claro y mantenible.
- Centraliza la configuración en src/config/env.js.
- Crea manejo de errores controlado con AppError.
- Agrega logs simples.
- Normaliza textos para comparar códigos de máquina y encabezados.
- La lectura del Google Sheet debe ser tolerante a filas vacías, encabezados y cambios en la cantidad de filas.
- El sistema debe continuar aunque un producto no exista en HubSpot.
- No detengas todo el proceso por un producto con error; agrega ese error al resumen.

Funciones esperadas en los servicios:

hubspot.service.js debería incluir funciones similares a:

- getDealLineItems(dealId)
- getMachineCodeFromLineItems(lineItems)
- findProductByPartNumber(partNumber)
- createMaintenanceLineItem({ dealId, product, row, frequencies, machineCode })
- getExistingMaintenanceLineItems(dealId)

Google Sheets service debería incluir funciones similares a:

- getSpreadsheetSheets()
- getSheetValues(sheetName)
- findSheetByMachineCode(machineCode)
- parseMaintenanceRows(values)

maintenance.service.js debe orquestar todo el flujo:

- validate dealId
- get deal line items
- extract machine code
- find matching Google Sheet
- parse rows
- check duplicates
- search product by part number
- create line item
- return summary
